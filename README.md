# 67164494-WORKSHOP1

## 1. บทนำและความเป็นมา (Introduction & Background)
ในยุคปัจจุบันที่เทคโนโลยีดิจิทัลเข้ามามีบทบาทสำคัญในชีวิตประจำวัน พฤติกรรมการเลือกซื้อสินค้าของผู้บริโภคได้เปลี่ยนผ่านจากหน้าร้านแบบดั้งเดิม (Physical Store) เข้าสู่ระบบออนไลน์อย่างเต็มรูปแบบ โดยเฉพาะอย่างยิ่งในกลุ่มธุรกิจแฟชั่นและเครื่องแต่งกาย ซึ่งถือเป็นหนึ่งในอุตสาหกรรมที่มีอัตราการเติบโตและมูลค่าการตลาดบนแพลตฟอร์มพาณิชย์อิเล็กทรอนิกส์ (E-Commerce) สูงที่สุด เนื่องจากเข้าถึงกลุ่มเป้าหมายได้หลากหลาย ทลายข้อจำกัดเรื่องเวลาและสถานที่ในการเลือกซื้อสินค้า

แบรนด์เสื้อผ้า **SALA (ศาลา)** เล็งเห็นความสำคัญของการเพิ่มขีดความสามารถในการแข่งขันบนตลาดออนไลน์ จึงได้เริ่มพัฒนาแนวคิดในการสร้างแพลตฟอร์ม E-Commerce ของตนเองขึ้นมา ทว่าการดำเนินธุรกิจในปัจจุบันไม่ได้จำกัดอยู่เพียงแค่การมีหน้าเว็บไซต์เพื่อแสดงสินค้าเท่านั้น แต่หัวใจสำคัญที่จะทำให้ธุรกิจเติบโตได้อย่างยั่งยืนคือ "ประสิทธิภาพในการบริหารจัดการหลังบ้าน" ร้านค้าแฟชั่นส่วนใหญ่มักประสบปัญหาการจัดการคลังสินค้าที่ไม่แม่นยำเนื่องจากเสื้อผ้ามีตัวแปรที่ซับซ้อน เช่น ขนาด (Size) สี (Color) และคอลเลกชัน รวมถึงการขาดเครื่องมือในการประมวลผลข้อมูลยอดขายที่จะนำมาวิเคราะห์พฤติกรรมลูกค้าได้อย่างเป็นรูปธรรม

ด้วยเหตุนี้ โครงงานนี้จึงจัดทำขึ้นภายใต้รายวิชา CSI204 (SUMMER SEMESTER 3/2568) เสนออาจารย์ทินภัทร บริรักษ์ เพื่อออกแบบและพัฒนาระบบเว็บแอปพลิเคชัน E-Commerce สำหรับแบรนด์ SALA STORE โดยมุ่งเน้นการมอบประสบการณ์การใช้งานที่ลื่นไหล ปลอดภัย และตอบโจทย์ผู้ใช้งานทั้ง 2 กลุ่มหลัก คือ ลูกค้า (Customer) ที่ต้องการระบบการเลือกซื้อสินค้าและชำระเงินที่สะดวกรวดเร็ว และผู้ดูแลระบบ (Admin) ที่ต้องการระบบควบคุมสต็อกสินค้าและหน้าจอรายงานสถิติ (Dashboard) ที่แม่นยำ เพื่อขับเคลื่อนธุรกิจด้วยข้อมูลและยกระดับแบรนด์ SALA สู่การเป็นผู้นำร้านค้าแฟชั่นออนไลน์ยุคใหม่ได้อย่างเต็มประสิทธิภาพ

## 3. ปัญหาที่พบและต้องการแก้ไข (Pain Points)
* **การจัดการสต็อกสินค้าผิดพลาด:** ร้านค้าทั่วไปที่ไม่มีระบบหลังบ้านที่ดี มักประสบปัญหาคำนวณจำนวนสต็อกเสื้อผ้า ไซส์ หรือสีผิดพลาด ทำให้ลูกค้ากดสั่งซื้อสินค้าที่ไม่มีอยู่จริงในคลัง
* **ข้อมูลยอดขายกระจัดกระจาย:** ข้อมูลการสั่งซื้อและรายรับไม่ได้ถูกรวบรวมอย่างเป็นระบบ ทำให้ผู้บริหารร้านวิเคราะห์แนวโน้มคอลเลกชันเสื้อผ้าที่ขายดีหรือคำนวณกำไร-ขาดทุนได้ยาก
* **ประสบการณ์การใช้งานไม่ลื่นไหล:** เว็บไซต์ขายเสื้อผ้าบางแห่งไม่ได้รองรับการแสดงผลบนสมาร์ตโฟน (Responsive) และมีขั้นตอนการหยิบของใส่ตะกร้าไปจนถึงการชำระเงินที่ซับซ้อนเกินไป

## 4. ที่มาและความสำคัญ (Rationale)
จากปัญหา (Pain Points) ที่ได้กล่าวไปข้างต้น แบรนด์ **SALA** จึงได้คิดค้นแพลตฟอร์มนี้ขึ้นเพื่อรวมระบบการเลือกซื้อสินค้าเสื้อผ้าแฟชั่นที่ใช้งานง่ายบนทุกอุปกรณ์ และหน้าจอสรุปข้อมูลสถิติสำหรับผู้ดูแลระบบ (Admin Dashboard) เข้าไว้ด้วยกัน เพื่อเพิ่มประสิทธิภาพในการบริหารคลังสินค้าและการตัดสินใจทางธุรกิจให้เบ็ดเสร็จในแพลตฟอร์มเดียว

## 5. วัตถุประสงค์ของโครงงาน (Objectives)
* เพื่อพัฒนาและออกแบบระบบเว็บแอปพลิเคชัน E-Commerce สำหรับร้านเสื้อผ้าแบรนด์ SALA ให้มีประสิทธิภาพและใช้งานง่าย
* เพื่อออกแบบและพัฒนาระบบหลังบ้านสำหรับ Admin ในการควบคุมสต็อกสินค้าเสื้อผ้าและจัดการหมวดหมู่ได้อย่างถูกต้อง
* เพื่อสร้างระบบรายงานและสรุปข้อมูล (Admin Dashboard) ที่ช่วยให้ผู้ดูแลระบบมองเห็นภาพรวมยอดขายและวิเคราะห์สินค้าขายดีได้อย่างแม่นยำ

## 6. ความต้องการของผู้ใช้งาน (User Requirements)
ระบบมีผู้ใช้งานหลัก 2 กลุ่ม คือ:
* **ลูกค้า (Customer)**
  * สมัครสมาชิก / เข้าสู่ระบบ
  * ค้นหาและเลือกซื้อสินค้าเครื่องแต่งกาย
  * เพิ่มสินค้าลงตะกร้า (Cart)
  * สั่งซื้อสินค้าและชำระเงินจำลอง (Checkout)
  * ดูประวัติการสั่งซื้อ (Buy History)
* **ผู้ดูแลระบบ (Admin)**
  * จัดการข้อมูลสินค้า (เพิ่ม / แก้ไข / ลบ)
  * จัดการหมวดหมู่สินค้า (Category)
  * จัดการผู้ใช้งานในระบบ และตรวจสอบข้อมูลลูกค้า
  * ดูรายงานสถิตียอดขายและข้อมูลวิเคราะห์ผ่าน Dashboard

## 7. ขอบเขตของระบบ (System Scope)
ระบบครอบคลุมฟังก์ชันหลัก 9 ส่วน ตามที่กำหนดไว้ในรายวิชา ได้แก่:
1. การจัดการสมาชิก (Register / Login)
2. การจัดการข้อมูลสินค้า (Product Management)
3. การค้นหาและแสดงรายละเอียดสินค้า (Search & View Products)
4. ระบบตะกร้าสินค้า (Shopping Cart)
5. ระบบสั่งซื้อสินค้า (Order Management)
6. ระบบชำระเงินจำลอง (Simulation)
7. ระบบติดตามสถานะคำสั่งซื้อ (Order Tracking)
8. ระบบจัดการสินค้าและคำสั่งซื้อสำหรับผู้ดูแลระบบ
9. รายงาน / Dashboard สรุปข้อมูลยอดขายเบื้องต้น
